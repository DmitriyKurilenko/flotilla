## 21.05.2026 — v0.1.0

### Новое
- Первый релиз flotilla. Один установщик одной командой поднимает на
  чистом сервере всё необходимое: Docker, общую сеть, обратный прокси с
  автоматическими SSL-сертификатами.
- Развёртывание проекта одной командой: достаточно положить рядом два
  файла (описание проекта и compose), флотилла остальное сделает сама.
- Для простых проектов (один сервис, один домен) сертификат и
  маршрутизация настраиваются автоматически — руками ничего прописывать
  не нужно.
- Команда статуса показывает по проекту одной сводкой: контейнеры,
  маршруты, сертификат и срок его действия, доступность по HTTPS,
  когда был последний деплой.
- Проверка «вхолостую» (`--dry`) — можно убедиться что проект корректен
  до фактического развёртывания, ничего не меняя на сервере.
- Перед каждым деплоем выполняются восемь проверок-предохранителей,
  каждый из которых добавлен после реальной аварии в проде.
- Три лёгких тестовых проекта из GitHub (echo-сервер, файловый менеджер,
  RSS-ридер) для проверки всех сценариев развёртывания.

### Исправления
- Установщик теперь обнаруживает чужой Traefik (без Docker-провайдера)
  и сообщает об этом сразу, вместо того чтобы молча пропускать шаг и
  получать ошибку при деплое проектов.
- Установщик теперь проверяет занятость порта 8080 (Traefik dashboard)
  ещё на этапе диагностики, чтобы не допустить непонятной ошибки
  «port already allocated» при запуске Traefik.
- Исправлена несовместимость Traefik с современными версиями Docker
  (25.0+), когда Traefik не мог подключиться к Docker-провайдеру из-за
  устаревшей версии API, и проекты оставались недоступными.
  Установщик теперь требует Docker 25.0+.

### Известно
- Установщик скачивает готовый бинарь только после публикации релиза;
  до этого момента сборка выполняется из исходников.
